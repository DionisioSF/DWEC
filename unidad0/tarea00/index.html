Compra de entradas para concierto

  1. Desde un cliente, accedes a la web de venta de entradas y pulsas sobre el botón del concierto que quieres asistir.
  2. Se lanza una consulta al servidor con la petición de la información del concierto.
  3. En el servidor se hará una busqueda de las entradas vendidas / disponibles y devolverá el resultado al cliente.
  4. En el cliente se mostrará el resultado de la petición, permitiendo seleccionar únicamente las disponibles.
  5. Una vez seleccionadas y pulsando sobre comprar, enviará una nueva petición al servidor para "marcar" esas entradas como "reservadas" para que durante un tiempo, unos 10 minutos por ejemplo, a otro cliente no le aparezcan disponibles y no pueda comprar las mismas, mientras completas el proceso de compra.
  6. En el cliente se comprobará si hay un login activo, de forma que si está activo te redigirá a la plataforma de pago, o si no está activo, te mostrará el formulario para que puedas introducir tu usuario / contraseña, o registrarte como nuevo usuario.
  7. Si introduces usuario y contraseña, se enviará una petición al servidor para buscar en la bbdd al usuario introducido, y en caso de que el usuario exista, comprobará si la contraseña es correcta.
  8. Si todo es correcto, devolverá la información del perfil del usuario y redigirá a la plataforma de pago.
  9. En caso de que el usuario no exista o la contraseña no sea correcta, devolverá un error / aviso.
  10. Si vas a registrarte como nuevo usuario, el cliente mostrará el formulario con los datos del perfil a completar y una vez completado y pulsado el botón correspondiente de registrar / enviar, enviará al servidor los datos recogidos del formulario, para crear una nueva entrada en la bbdd.
  11. Una vez creado el nuevo usuario en la bbdd, el servidor devolverá los datos del perfil al cliente, para que pueda completar el proceso de compra, redirigiéndolo a la plataforma de pago.
  12. Si durante este proceso a pasado más tiempo del estipulado, en este caso 10 minutos, o cierras el navegador, el cliente enviará una petición el servidor para "liberar" esa reserva de entradas y que vuelvan a estar disponible.
